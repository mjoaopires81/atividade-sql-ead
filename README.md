# Atividade prática - SQL com GitHub em Educação a Distância
Este repositório foi criado para ilustrar o uso do **GitHub** como media digital de apoio à aprendizagem da linguagem **SQL** em contexto de Educação a Distância (EaD).

A atividade faz parte de uma proposta pedagógica em que os alunos:
- exploram diferentes media digitais (GitHub, vídeos tutoriais, W3Schools, simulador SQL, IA);
- desenvolvem consultas em SQL a partir de uma base de dados de exemplo;
- registam o seu progresso através de *commits*;
- analisam erros e utilizam a IA de forma ética para compreender e melhorar o código.

---

## Estrutura do Repositório

- `atividade/`
  - `enunciado_atividade_sql.md` – descrição da tarefa proposta aos alunos;
  - `base_dados.sql` – ficheiro com a base de dados de exemplo.

- `exemplos_erros/`
  - `consulta_errada.sql` – exemplo de consulta incorreta;
  - `consulta_corrigida.sql` – versão corrigida da consulta;
  - `explicacao_erro.md` – breve explicação do erro e da correção.

- `reflexao/`
  - `exemplo_reflexao_aluno.md` – exemplo de resposta à pergunta “Que media foi mais útil para aprender SQL e porquê?”.

---

## Orientações para os estudantes

1. Aceder ao repositório e fazer *fork* (ou aceitar a assignment no GitHub Classroom, se for o caso).
2. Importar o ficheiro `base_dados.sql` para o simulador SQL ou para o XAMPP.
3. Implementar as consultas pedidas no `enunciado_atividade_sql.md`.
4. Registar o progresso através de *commits* com mensagens claras (por exemplo: `cria tabela clientes`, `adiciona consulta com WHERE`).
5. Quando surgirem erros:
   - tentar primeiro identificar e corrigir o problema;
   - se necessário, utilizar uma ferramenta de IA para explicar o erro **e depois validar a resposta**;
   - registar a correção num novo *commit*.
6. No final, escrever uma breve reflexão sobre:
   - que media foi mais útil (GitHub, vídeo, W3Schools, IA, etc.);
   - que dificuldades surgiram e como foram ultrapassadas.

---

## Uso ético da IA

Os alunos podem utilizar ferramentas de IA:
- para esclarecer dúvidas;
- para compreender mensagens de erro;
- para comparar abordagens.

**Não devem:**
- submeter código gerado integralmente pela IA sem compreender o que faz;
- copiar respostas sem validação crítica.

A IA é um apoio à aprendizagem, não um substituto do raciocínio.
